# 시즌 일정 그룹화 (진행중 / 예정 / 완료)

## 배경

현재 "시즌 일정" 카드는 모든 그랑프리를 하나의 테이블에 나열하고, 각 행마다 "완료"/"예정" 텍스트만 표시한다. 그랑프리 수가 늘어날수록 스크롤이 길어지고 지난 라운드를 계속 봐야 하는 불편함이 있다.

## 목표

그랑프리를 **진행중 / 예정 / 완료** 세 그룹으로 나누어 표시하고, 완료된 그랑프리는 기본적으로 접어서 요약만 보이게 한다.

## 그룹 판정 기준

라운드별로:
- **주말 시작 시각** = 해당 라운드의 모든 세션(FP1/FP2/FP3/스프린트 예선/스프린트/예선/레이스) 중 가장 이른 시각
- **주말 종료 시각** = 레이스 세션 시각 (기존 `raceDt`)

판정:
- `now < 주말 시작` → **예정**
- `주말 시작 ≤ now ≤ 주말 종료` → **진행중**
- `now > 주말 종료` → **완료**

## 화면 구성

기존 "시즌 일정 (2026)" 카드 내부, 위에서 아래 순서:

1. **진행중** — 해당 그랑프리가 있을 때만 표시(보통 0~1개). 기존 강조 스타일(`next-race`류) 유지.
2. **예정** — 기존과 동일한 테이블. `NEXT` 배지는 다음 세션이 속한 그랑프리에 그대로 유지.
3. **완료 (N개)** — `<details>`로 기본 닫힘. 요약 텍스트에 완료 개수 표시. 펼치면 기존과 동일한 테이블.

각 그랑프리 행 안쪽에 있던 "세션별 KST 시간 보기" `<details>`(그랑프리 단위 세션 상세)는 그대로 유지된다. 이번 변경은 그 바깥에 그룹 구분 레이어를 추가하는 것이다.

## 구현 범위

- `index.html`의 `renderSchedule(now)` 함수만 수정
- 그랑프리 한 건을 렌더링하는 로직을 헬퍼 함수로 추출해 세 그룹에서 재사용 (중복 제거)
- 새 파일/외부 의존성 없음, 기존 데이터 흐름(`races`, `allSessions`) 그대로 사용

## 테스트 방법

- 로컬에서 index.html을 열어 현재 데이터 기준으로 세 그룹이 올바르게 나뉘는지 확인
- 그랑프리가 하나도 진행중이 아닌 일반적인 상황과, 레이스 위크엔드 중(진행중 그룹 존재) 상황을 각각 확인 (날짜를 임시로 조작해 테스트)
- "완료" details가 기본 닫힘 상태로 렌더링되는지, 클릭 시 펼쳐지는지 확인
